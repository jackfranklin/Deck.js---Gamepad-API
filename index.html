What is a JS Design Pattern?

A pattern is a reusable solution that can be applied to commonly occurring problems in software design
Addy Osmani, Learning JS Design Patterns

Why?

  • Better organised code
  • More maintainable code
  • Seperation of functionality into independent modules - reuse

Anti-Patterns

  • Code is not DRY
  • Pollutes the (global) namespace
  • Prototyping Object
  • Inline JS
  • Tightly coupled modules

Module Pattern


        var counter = {
          count = 0,
          addCount = function() {
            return ++count;
          },
          resetCount = function() {
            count = 0;
            return 0;
          },
          getCount = function() {
            return count;
          }
        }

Module Pattern


        counter.addCount() //1
        counter.getCount() //1
        counter.count = 4; //we don't want them to do this!

Module Pattern

How can we prevent users getting to the count variable directly & only editing it through our methods?

JavaScript doesn't provide private variables but we can emulate them.

Module Pattern


        var counter = (function() {
          var count = 0;
          var addCount = function() {
            return ++count;
          };
          var getCount = function() {
            return count;
          };
          return {
            addCount: addCount,
            getCount: getCount
          };
        })();

Module Pattern


        counter.addCount(); //1
        counter.getCount(); //1
        counter.count = 4; //undefined
